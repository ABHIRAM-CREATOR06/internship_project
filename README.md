## What is this?

BankPulse takes a loan applicant's financial profile, runs it through a trained ML model, and returns a clear **Approved / Rejected** decision alongside a human-readable explanation generated by Mistral AI — no jargon, no black box.

Built as a full-stack internship project: training notebook → serialised model → Flask API → vanilla JS frontend.

---

## Features

- **Loan prediction** — trained Random Forest model on a labelled loan approval dataset
- **AI explanation** — Mistral LLM explains the decision in plain language via `/assistant`
- **Dataset stats** — `/stats` endpoint surfaces live aggregates from the CSV
- **Health check** — `/health` for quick service status
- **Smoke test** — one-command validation of all API routes
- **Secure by default** — API keys stay server-side; never exposed to the browser

## Quickstart

### 1. Clone & create a virtual environment

```bash
git clone https://github.com/ABHIRAM-CREATOR06/internship_project.git
cd internship_project

python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure environment variables

```bash
cp .env.example .env
```

Open `.env` and fill in your values:

```env
MISTRAL_API_KEY=your_mistral_api_key_here
MISTRAL_MODEL=mistral-small-latest
MISTRAL_MAX_TOKENS=300
MISTRAL_TEMPERATURE=0.4
MISTRAL_TIMEOUT_SECONDS=10

MODEL_PATH=loan_approval_model.pkl
SCALER_PATH=scaler.pkl
DATASET_PATH=loan_approval_dataset.csv
```

> **Note:** The Mistral API key is read only by `backend/app.py` on the server and is never sent to the browser.

### 4. Run the app

```bash
python backend/app.py
```

Open **http://127.0.0.1:5000** in your browser.

## API Reference

| Method | Route | Description |
|--------|-------|-------------|
| `GET` | `/health` | Service health check |
| `GET` | `/stats` | Descriptive stats from the dataset |
| `POST` | `/predict` | Run loan approval inference |
| `POST` | `/assistant` | Get a plain-English explanation from Mistral |

### `POST /predict` — example request

```json
{
  "no_of_dependents": 2,
  "education": "Graduate",
  "self_employed": "No",
  "income_annum": 600000,
  "loan_amount": 1500000,
  "loan_term": 12,
  "cibil_score": 720,
  "residential_assets_value": 2000000,
  "commercial_assets_value": 500000,
  "luxury_assets_value": 300000,
  "bank_asset_value": 800000
}
```

### `POST /predict` — example response

```json
{
  "decision": "Approved",
  "confidence": 0.91
}
```

## Model Details

| Attribute | Value |
|-----------|-------|
| Algorithm | Random Forest Classifier |
| Dataset | `loan_approval_dataset.csv` |
| Preprocessing | MinMaxScaler on continuous features |
| Serialisation | `joblib` (`.pkl`) |
| Notebook | `loan_approval_predictor.ipynb` |

The full training pipeline — EDA, feature selection, model training, and evaluation — is documented in the Jupyter notebook.
